# Output Language Rules

This rule file governs language selection for all ARS Antigravity agent output. Antigravity reads `.agent/rules/` at every conversation; these rules are always-on.

Canonical source: `/AGENTS.md` §Iron Rules — Rule 8 ("Output language follows the user").

---

## Rule 1 — Chat language follows the user

Match the language the user writes in. If the user writes in Russian, respond in Russian. If the user switches languages mid-conversation, switch with them.

**Default for this workspace:** Russian for conversational responses and status messages.

---

## Rule 2 — Academic terminology stays canonical

Academic terms, methodology labels, statistical concepts, journal names, citation formats, and discipline-specific vocabulary remain in their canonical language regardless of the interface language. Do not translate "PRISMA," "IMRaD," "FINER," "RoB 2," "GRADE," "APA 7.0," "p-value," "effect size," or equivalent technical terms.

**Examples:**
- Russian interface: "Используем критерии FINER для оценки исследовательского вопроса."
- Russian interface: "Оценка риска смещения по шкале RoB 2 показала…"
- English technical term in a Russian sentence is correct and expected.

---

## Rule 3 — Reports inherit the manuscript's language

Research reports, paper drafts, review reports, editorial decision letters, and revision roadmaps are produced in the **manuscript's language**, not the interface language — unless the user explicitly overrides.

**Example:** If the user writes in Russian but the paper being reviewed is in English, the review reports are in English. The status message presenting those reports may be in Russian.

**Override:** User may say "produce the review in Russian" or "give me the roadmap in English" — respect that override for the remainder of the session or until told otherwise.

---

## Rule 4 — Bilingual abstracts are always bilingual

The `academic-paper` skill's Phase 5b (`@abstract-bilingual`) always produces abstracts in both the paper's primary language and English — or both languages specified in the Paper Configuration Record. This is not optional and is not subject to the "follow the user" rule for that particular deliverable.

**Bilingual abstract quality standard:** Each abstract is independently composed, not a mechanical translation. Both versions cover the same key points in the same order. Terminology is consistent across both.

---

## Rule 5 — Process summary language is negotiated

The Stage 6 Process Summary (generated by @orchestrator in the `academic-pipeline`) is produced in the user's language by default. Before generating, @orchestrator asks: "Which language version do you prefer for the Process Summary?" If no answer is given, use the current interface language.

---

## Rule 6 — MCP notices and error tokens are always in English

System-level tokens (`[REQUIRES CLARIFICATION]`, `[UNVERIFIED CITATION — NO QUOTE OR PAGE LOCATOR]`, `[MCP UNAVAILABLE: <tool>]`, `[GENERATOR-PHASE-ABORTED: …]`, `[PRE-COMMITMENT-ACKNOWLEDGED]`) are always emitted in English, regardless of the interface language. These are machine-readable markers; their language is fixed.

---

## Language Coverage

This workspace is maintained by a Russian-speaking maintainer (sipki-tech). The agent team is designed to serve researchers who may work in Russian, English, or both simultaneously. The upstream ARS source (Cheng-I Wu) used Traditional Chinese (zh-TW) and English; those examples are preserved in the vendored source but do not set the default for this edition.

If a user supplies text in a language other than Russian or English, respond in that language for conversational turns; for technical deliverables, default to English and confirm with the user if in doubt.
